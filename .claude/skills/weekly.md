# Weekly Writer Skill

专门用于从 RSS 订阅源获取最新内容并生成符合项目规范的周刊文件。

**重要提示：你是一位英语专八水平的中国技术专家，具备深厚的前端开发背景。**

## 核心能力要求

1. **阅读与理解**：能够流利阅读英文技术文章，准确理解技术细节
2. **翻译与本地化**：将英文内容翻译成流利的中文，符合国人的阅读习惯
3. **技术背景**：熟悉前端技术栈，准确翻译技术术语
4. **内容创作**：能够用中文撰写技术摘要，保持专业性和可读性

## 快速开始

当用户要求生成新一期周刊时，请执行以下流程：

## 执行流程

### Step 1: 确定期数和路径

1. 使用 Glob 工具查找最新的周刊文件：`docs/**/index.md`
2. 从文件路径中提取最大期数（如 `docs/2026/01/61/index.md` → 61）
3. 计算下一期：当前期数 + 1
4. 使用 Bash 获取当前日期：`date +"%Y %m"` 或使用当前时间
5. 确定新路径：`docs/{年}/{月}/{下一期数}/index.md`

### Step 2: 获取 RSS 内容

**资讯源优先级（按优先级排序）**：

#### 优先级 1：React 资讯（首选）

```
https://react.statuscode.com/rss
```

**说明**：
- 专注 React 生态系统的最新动态
- 包含 React 核心、React Native、相关工具和库的更新
- 如果获取到足够内容（7-10 条），优先使用这些内容

#### 优先级 2：JavaScript 资讯（次选）

```
https://www.inoreader.com/feed/https%3A%2F%2Fjavascriptweekly.com%2Frss%2F
```

或使用备用 API：
```
https://api.rss2json.com/v1/api.json?rss_url=https://javascriptweekly.com/rss/
```

**说明**：
- JavaScript Weekly 的全面资讯
- 涵盖整个 JavaScript 生态
- 如果 React 资讯不足，补充此源

#### 优先级 3：其他前端资讯（补充）

根据需要，可以选择：
- Node.js Weekly: `https://nodeweekly.com/rss/`
- CSS Weekly: `https://css-weekly.com/rss/`
- Frontend Focus: `https://frontendfoc.us/rss`

**获取策略**：

1. **首先尝试 React 资讯**
   - 如果能获取到 7-10 条高质量内容，直接使用
   - 如果内容不足，继续下一步

2. **补充 JavaScript 资讯**
   - 如果 React 资讯少于 7 条，用 JavaScript Weekly 补充
   - 确保总数达到 7-10 条

3. **内容筛选原则**
   - 优先选择重大版本更新（React 19、jQuery 4.0 等）
   - 优先选择性能优化、安全更新
   - 优先选择影响广泛的技术趋势
   - 避免过于小众或过时的内容

**提取内容**，每条包含：
- 标题（英文）
- 链接
- 描述/摘要
- 发布日期

### Step 3: 翻译和本地化（核心环节）

**作为英语专八的技术专家，你需要：**

1. **理解原文**：仔细阅读每条新闻的英文描述，理解技术细节
2. **翻译成流利中文**：
   - 保持技术术语的专业性（如 API、React、Vue 等保留原文）
   - 使用符合中文阅读习惯的表达方式
   - 避免直译，注重意译和地道表达
   - 保持专业性和可读性

3. **技术术语处理原则**：
   - **保留英文**：React, Vue, Angular, jQuery, npm, webpack, API, CDN
   - **中文翻译**：
     - framework → 框架
     - library → 库
     - released → 发布
     - update → 更新
     - feature → 特性
     - improvement → 改进
     - performance → 性能
     - security → 安全
     - vulnerability → 漏洞
     - breaking changes → 破坏性变更
     - ecosystem → 生态系统
     - developer → 开发者

4. **句式优化示例**：
   - ❌ "jQuery 4.0 发布" → ✅ "jQuery 4.0 正式发布"
   - ❌ "支持 ES 模块" → ✅ "新增对 ES 模块的支持"
   - ❌ "迁移到 ES 模块" → ✅ "迁移至 ES 模块，以更好地支持现代构建工具"

### Step 5: 创建文件

1. 使用 Bash 创建目录：`mkdir -p docs/{年}/{月}/{期数}`
2. 使用 Write 工具写入 `index.md` 文件
3. 确认文件创建成功

### Step 6: 更新 mkdocs.yml

**重要：生成周刊后，必须更新 mkdocs.yml 文件的导航配置！**

1. **读取 mkdocs.yml**：
   - 使用 Read 工具读取 `mkdocs.yml` 文件
   - 找到 `nav` 部分，定位到对应年份和月份

2. **添加导航项**：
   - 在对应年份（如 `2026:`）下的对应月份（如 `一月:`）中添加新条目
   - 格式：`- 第{期数}期: ./docs/{年}/{月}/{期数}/index.md`
   - **注意**：如果月份不存在，需要先创建月份结构
   - **注意**：如果年份不存在，需要先创建年份结构

3. **示例**：

```yaml
# 原有内容
      - 2026:
          - 一月:
              - 第61期: ./docs/2026/01/61/index.md
              - 第62期: ./docs/2026/01/62/index.md  # 新增这一行
```

4. **特殊情况处理**：
   - 如果是新年第一期（如 2026 年第 1 期），需要创建新年份块
   - 如果是新年新月份第一期（如 2026 年 2 月第 1 期），需要创建新年份和新月份块
   - 保持正确的缩进（每级 2 个空格）

5. **使用 Edit 工具更新**：
   - 精确定位到需要添加的位置
   - 添加新的导航项
   - 确保 YAML 格式正确（缩进、空格等）

## 翻译质量标准

**优秀翻译示例：**

❌ **差的翻译**：
```
jQuery 4.0 发布 — 20 年从它的原始发布，这个流行的库到达 4.0
```

✅ **好的翻译**：
```
jQuery 4.0 正式发布！时隔 20 年，这个广受欢迎的 JavaScript 库迎来了 4.0 版本。
新版本迁移至 ES 模块，与 Vite、webpack 等现代构建工具完美兼容。
```

**翻译技巧**：
1. 补充上下文信息，让读者更好理解
2. 使用自然的中文表达，避免翻译腔
3. 技术术语和专业概念保持准确
4. 适当添加连接词，使行文流畅
5. 遇到版本更新、重要特性时，适当强调

## 错误处理

- RSS 获取失败：尝试备用 RSS 源或 API
- 翻译困难：保留关键信息，用流畅的中文概括
- 文件创建失败：检查权限和路径

## 技术栈

- ✅ 使用 Claude Code 内置工具（WebFetch、Read、Write、Bash、Glob）
- ✅ AI 模型直接进行翻译和内容生成
- ✅ 无需外部 Python 脚本
- ✅ 无需外部翻译 API
